На самом деле с программой мы взаимодейтсвуем передовая ей аргументы(как в той выше программе) и получая(возвращая) от нее значения(return) например как в прошлой программе. 0 если все ок.

НО! вспомните, вам когда-нибудь реально реально нужно было от программы ее return? Когда вы в последний раз передовали программе аргументы?

На самом деле основной способ взаимодейтсвия с программой: поток ввода или stdin(в c это stdio). Мы это вводим в программу.

Есть два стандратных потока вывода. Собственно стандратный поток вывода(stdout), и стандратный поток вывода ошибок(stderr).

Строго говоря три файловых дескриптора с номерами 0, 1, 2 соответственно

В C для ввода изпользуется scanf(), вывода printf(), а для ошибок учитель не помнит.

В C++ мы итак знаем, но для ошибок: cerr()

МЫ очевидно получив что-то потоком ввода, должны это сохранить и обработать это. А сами типы данных в c++ работает не так как в python. Другой тип модели данных.

Дело в том что в c++ все сохраняется в реальные переменные(в отличии от python). Компьютер выделяет программе ресурс: оперативную память. А переменные в c++ это ссылки на конкретную ячейку в памяти(состояющию из байт). А переменные строго-типизированные. У переменных есть реальные ячейки в памяти.

ПОЧЕМУ "Реальная переменная"? Начнем с того что у вас есть три ячейки памяти, в которых хранится 0 или 1. Наименьшее число 000. Давайте будем его инкрементировать(увеличивать на 1):

000
001
010
011
100
101
111

А на этом все, больше инкрементировать нельзя 111 - самое большое двоичной трехзначное число. После инкремента последует 000 и по кругу. Аналогия с часами: 23:59 и потом 0:00. Это называется кольцо вычетов.

Это все к тому что реальные переменные они ограниченны в размере. Фиксированный размер количество байт. Собственно, минимальный размер нашей переменной это 1 байт(8 бит), при чем скорее всего чтобы поддерживать еще и отрицательные числа мы можем сделать первую цифру(1 или 0) индикатором, если 0 - то число положительно, а 1 - число отрицательное.

ТИПЫ ДАННЫХ:

1. short int - 2 байт - 16 бит
2. int или long int - 4 байт - 32 бит
3. long long int - 8 байт - 64 бит

собственно надо помнить что первая цифра это индикатор + или -, и у нас собственно остается n - 1 бит на запоминание числа, и так как у нас двоичная система, диапозон чисел которые мы можем запомнить таков:

1. short - от -2^15 до 2^15 - 1(из-за того что ноль тоже положительное)
2. int - от -2^31 до 2^31 - 1
3. lon long - от -2^63 до 2^63 - 1


Есть типы float(32 бит) и double(64 бит). Предпологается что в связи с тем что float часть битов уходит на знаки после запятой, а часть на до запятой, то не получается сделать большой диапозон того какие числа могут быть записанны. ПОэтому рекомендуется изпользовать double.

## Ход исполнения программы

"Вечный цикл не всегда вечный кайф"

Вот древний способ вечного цикла, goto - go to инструкция прямого перехода по ссылке(метке)

```c++
#include <iostream>
using namespace std;
int main() {
 loop:
    cout << "Что-то" << endl;
    goto loop;
    return 0;
}
```

Строго говоря, мы можем это изпользовать для того чтобы вырватся из цикла:

```c++
#include <iostream>
using namespace std;

int main()
{
    int x = 0;
loop:
    if (x >= 10) goto out;
    cout << x << "\n";
    x++;
    goto loop;
out:
    return 0;
}
```

Еще, сказали что для олимпиадного программирования быстрее выполняется работа с диском при `\n` чем при endl, хоть endl и красивее пишется.

Алрогитм Евклида - позволяет вычеслять Наибольший общий делитель, не изпользую даже операцию деления а лишь вычитаение. Это пример работы с if и while

```c++
#include <iostream>
using namespace std;

int main()
{
    long long int a, b;
    cin >> a >> b;
    while (a != b) {
        if (a > b) {
            a -= b;
        } else {
            b -= a;
        }
    }
    cout << a << "\n";
    return 0;
}
```
